Цель создать поступающие сырые данные с максимальной скоростью в прямом эфире к папке "стратегия" - где просто обычной логикой в Js коде расписана стратегия и она отрабатывает. То есть в любой момент на основе этого с ии за пару минут можно создать любую свою стратегию


КАК? data - собирает сырые данные - их сразу получает core (шина) - и направляет всем кто подписан на нужные данные, то есть стратегия - она просит eth_price как только поступили данные шина сразу отдаст ей их.

data - откуда берём данные и их получение

core - как эти данные передаются внутри бота (максимально эффективно)
